# v0.5.0 — DWG 渲染管线重整

## Context

对标浩辰CAD看图王，重新梳理 DWG→渲染 的完整链路。当前 DWG 渲染只有绿色框线，
而 DXF 渲染之前是正确的。v0.5.0 聚焦：**高渲染精度和还原度**，速度次要，防内存泄漏。

**约束**: 商业产品，MIT/BSD 许可证，速度不要求但精度要高，防内存泄漏。

---

## 管线架构

```
DWG 二进制文件
  ↓ [dwg_parser.cpp] R2004+ 解密+解压+对象映射
  ↓ [dwg_reader.cpp] LZ77解压+位流读取
  ↓ [dwg_objects.cpp] 逐实体位流解析 (17种类型)
  ↓
SceneGraph (std::variant 实体数组 + 块/图层表)
  ↓ [render_batcher.cpp] 颜色解析 + 曲面细分 + INSERT展开 + 批次合并
  ↓
RenderBatch 数组 (topology + color + vertex_data)
  ↓ [test_render_export.cpp] JSON 序列化 + gzip
  ↓
preview.html (Canvas 2D 渲染)
```

DXF 管线共享 SceneGraph 之后的全部链路，DXF 解析比 DWG 简单 5-10 倍且渲染结果正确。

## 问题诊断

### 问题 1: 颜色解析错误（最关键 — 导致 89% 绿色）

DWG R2004+ ENC (Entity Color) 和 CMC (Common Color) 解析都**丢弃了 RGB True Color 值**。

- `dwg_parser.cpp:1391` — `(void)reader.read_bl(); // rgb` 实体 RGB 被丢弃
- `dwg_reader.cpp:514` — `(void)read_bl(); // rgb` CMC RGB 被丢弃

导致 89% 顶点变成 (0,165,82) 绿色，而非参考图的灰/白/棕/绿/暗灰。

### 问题 2: BLOCK base_point 解析失败

R2010 BLOCK 实体的 `base_point` 字段读取 mostly NaN。已有 world-space identity transform workaround。

### 问题 3: HATCH 边界解析部分失败

84/15497 LWPOLYLINE 失败，9/1368 HATCH 失败。

### 问题 4: 5% 顶点在 Y≈0（原点 cluster）

可能解析错误或坐标系问题。fitView IQR 算法能排除。
