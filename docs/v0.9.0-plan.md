# v0.9.0 结构数据对齐里程碑计划

**日期**: 2026-05-03  
**版本目标**: 0.8.x → 0.9.0  
**核心主题**: 建立自研 DWG 解析器与 HOOPS Exchange SDK 参考数据之间的完整结构对齐诊断体系

---

## 背景

当前 DWG 解析器在基础几何实体（LINE、ELLIPSE、SPLINE）层面已达到约 95% 的解析准确率，但与 HOOPS Exchange SDK 参考数据对比时，块边界检测、图纸空间推断和场景树层级等结构性差距始终存在。这些结构性错位直接影响渲染正确性：块定义中的实体泄漏到顶层、INSERT 变换未正确应用、布局/视口语义不完整。

SCS 元数据提取（`extract_scs_tree.py`）可用于布局分组级别验证，但仅有约 18 个高层节点，没有实体级细节。实体级对比必须使用 HOOPS JSON 导出（`tools/dwg_to_json/`）。

本里程碑将对比工具链从简单的实体计数匹配升级为完整的结构对齐诊断体系，再利用这些诊断结果驱动针对性的解析器修复。

---

## 目标

1. **统一结构对比工具**：按维度输出通过/警告/失败（实体计数、空间匹配、块层级、空间归属、布局/视口、树拓扑）。
2. **增强 HOOPS 参考导出**：捕获块实例、变换矩阵和填充后的包围盒。
3. **增强自研解析器导出**：增加 `--compare-mode` 模式，输出扁平实体数组 + INSERT 展开元数据，以便与 HOOPS 公平对比。
4. **解析器修复**：解决阻碍对齐的前 3 大结构性差距：
   - 块边界检测遗漏（AM_5 泄漏、DEFPOINTS POINT 溢出）
   - BLOCK_HEADER 匿名名称截断（*D vs *D1077）
   - 图纸空间推断启发式规则替换

---

## 任务（按执行顺序）

### T1 — 统一对比脚本 `scripts/compare_structural.py`（新建）

将 `compare_hoops.py` + `compare_scene_tree.py` 的能力合并扩展为单一工具。

**对比维度及容差：**

| 维度 | 容差 | 优先级 |
|-----------|-----------|----------|
| 总实体数（模型空间） | ±5% | 关键 |
| 分类计数（GEOM_CURVE、POLYBREP、ANNOTATION、HATCH） | ±5% 或 ±2 个 | 关键 |
| 图层名称匹配 | 集合完全相等 | 关键 |
| 几何匹配率（质心匹配） | ≥ 95% | 关键 |
| 块定义数量 | 完全相等 | 高 |
| 块实例数量 | ±5% | 高 |
| 模型/图纸空间拆分比例 | ±10% | 中 |
| 布局数量 | 完全相等 | 中 |
| 视口数量 | 完全相等 | 中 |
| 变换对比（缩放/旋转/位置） | 缩放 1%、旋转 0.1 弧度、位置 1% 对角线 | 中 |
| 树深度 | ±1 | 低 |

**输出**：结构化 JSON 报告 + 人类可读文本。退出码：0=通过、1=警告、2=失败。

**复用现有代码**：
- 现有脚本中的 `load_json()`（UTF-8/GB2312 降级）
- `compare_scene_tree.py` 中的类型标准化表
- `compare_hoops.py` 中的空间匹配算法
- `compare_scene_tree.py` 中的包围盒偏差计算

### T2 — 增强 HOOPS 参考导出

**文件：** `tools/dwg_to_json/entity_visitor.cpp`

1. **增加 `insert_instances` 数组** — 对于 HOOPS 树中检测到的每个 INSERT，输出：`block_name`、`insertion_point`、`scale`、`rotation`、`layer`。HOOPS 将 INSERT 展平为实体，需要从 `A3DDrawingBlock` 遍历中恢复实例元数据。
2. **填充树节点包围盒** — 遍历过程中从实体几何计算 `bbox_min`/`bbox_max`，而不是保留在 (0,0,0)。
3. **增加 `space` 标注** — 确保每个实体都有显式的 `space`：`model`、`paper`、`annotation`。
4. **增加 `block_entity_counts`** — 每个块的实体数量，用于块层级验证。

**文件：** `tools/dwg_to_json/entity_visitor.h`
- 在 `EntityVisitor` 中增加 `insert_instances` 向量和 `block_entity_counts` 映射。

### T3 — 增强自研解析器导出

**文件：** `core/test/test_render_export.cpp`

1. **增加 `--compare-mode` 命令行开关** — 开启时输出新的顶层 `entities` 数组，包含每个实体的元数据（类型、图层、空间、包围盒、inBlock、blockIndex、属性）。正常导出保持不变，避免预览 JSON 膨胀。
2. **增加 `--expand-inserts` 命令行开关** — 开启时同时输出 INSERT 实体和展开的块实体（展开项标记 `fromInsert: true`）。这样可以与展平块的 HOOPS 进行公平的一对一对比。
3. **增加 `inserts` 数组** — 块实例元数据：`blockName`、`blockIndex`、`insertionPoint`、`scale`、`rotation`、`layer`。
4. **修复 `worldBounds` 填充** — 确保 `build_scene_tree()` 为有实体的所有节点计算非空包围盒。

### T4 — 解析器修复（由对比结果驱动）

**T4.1 — 修复 BLOCK_HEADER 匿名名称截断**

**文件：** `core/src/parser/dwg_table_prescan.cpp`、`core/src/parser/dwg_post_processing.cpp`

- 根因：对 BLOCK_HEADER 调用 `r.read_t()` 时，匿名尺寸块读取到的是 `*D` 而不是 `*D1077`。
- 修复：读取 BLOCK_HEADER 表对象时保留包含数字后缀的完整名称。优先使用 `block_names_from_entities`（来自 BLOCK 实体句柄流的正确完整名称）作为主数据源；仅当基于句柄的查找失败时才回退到 `m_sections.block_names`。
- 在 `core/include/cad/scene/block.h` 的 `BlockDefinition` 中增加 `dwg_full_name` 字段用于调试。

**T4.2 — 修复 Mechanical 块边界检测**

**文件：** `core/src/parser/dwg_block_tracking.cpp`、`core/test/test_render_export.cpp`

- 问题：`Drawing2.dwg` 中 AM_5 图层有 33 个 CIRCLE + 33 个 TEXT 从块定义泄漏。
- 修复：确保 `test_render_export.cpp` 中的 `block_entity_indices` 集合在渲染前同时包含 `entity_indices` 和 `header_owned_entity_indices`。将块合并索引集合中的所有实体都标记为 `in_block = true`。

**T4.3 — 修复 *D 块的 POINT 溢出**

**文件：** `core/src/parser/dwg_post_processing.cpp`

- 问题：`big.dwg` 中匿名 `*D` 尺寸块在 `DEFPOINTS` 图层上有 2622 个额外 POINT。
- 修复：后处理遍历块父链时，如果实体图层是 `DEFPOINTS` 且其所有者句柄匹配尺寸块，则无论是否有 BLOCK/ENDBLK 包围，都强制 `in_block = true`。

**T4.4 — 替换图纸空间推断启发式规则**

**文件：** `core/src/parser/dwg_post_processing.cpp`

- 当前启发式：`total_entities > 2` 就将块标记为图纸空间。这会误分类小型模型块和大型尺寸块。
- 修复：检查块的 BLOCK_HEADER 句柄是否被 LAYOUT 对象的所有者句柄引用。如果是，则确定性地标记为图纸空间。仅在确定性规则无法判断时，才将实体数量启发式作为带诊断标签的回退。
- 取消对推断为图纸空间的块的 `entity_indices`/`header_owned_entity_indices` 清除操作。改为仅在确定性规则确认后，才将其实体以 `in_block = false` 重新分配到图纸空间。

### T5 — 批量对比集成

**文件：** `scripts/batch_compare.py`

- 增加 `--structural` 模式，运行 `compare_structural.py` 替代旧版 `compare_hoops.py`。
- 跨所有 DWG 测试文件生成汇总矩阵。
- 增加 CI 友好的退出码。

## 验证流程

```bash
# 1. 构建
$ cd build && cmake --build . --target render_export && cmake --build . --target dwg_to_json

# 2. 生成 HOOPS 参考数据
$ ./build/tools/dwg_to_json/dwg_to_json test_dwg/Drawing2.dwg /tmp/Drawing2_hoops.json

# 3. 以对比模式生成本地导出
$ ./build/core/test/render_export test_dwg/Drawing2.dwg /tmp/Drawing2_ft.json --compare-mode

# 4. 运行结构对比
$ python3 scripts/compare_structural.py /tmp/Drawing2_hoops.json /tmp/Drawing2_ft.json --verbose

# 5. 批量运行所有测试文件
$ python3 scripts/batch_compare.py test_dwg --structural
```

## 风险及应对

| 风险 | 应对措施 |
|------|------------|
| HOOPS API 无法干净暴露 INSERT 变换 | 记录 API 限制；对块实例回退到仅实体计数对比 |
| 解析器修复引入 DXF 回归 | 修复前后运行完整 DXF 回归；以 DXF 通过为门禁 |
| `--compare-mode` 导致导出膨胀 | 仅在对比模式下输出 `entities` 数组；正常预览 JSON 不变 |
| 匿名名称修复需要句柄流变更 | 隔离到 BLOCK_HEADER 解析路径；解析失败时增加诊断 |
| 确定性图纸空间规则遗漏边界情况 | 保留启发式作为带标签回退；用测试文件矩阵调优 |

## 备注

- `scripts/compare_hoops.py` 和 `scripts/compare_scene_tree.py` 在 `compare_structural.py` 稳定后应标记为废弃。可改为输出废弃警告并重定向到新工具。
- `extract_scs_tree.py` 在布局分组级别验证中仍有价值，但不是 v0.9.0 的主要对比载体。
- 所有解析器修复必须是通用算法改进（遵循 v0.10.0 通用性原则），不得针对特定测试文件做特殊处理。
